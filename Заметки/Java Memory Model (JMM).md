# Java Memory Model (JMM)
Java Memory Model - это спецификация, которая описывает, как потоки взаимодействуют через память. Она задает правила: какие изменения, сделанные одним потоком, будут видны другим, управляет кэшированием, переупорядочиванием инструкций, видимостью переменных.
## Основные гарантии JMM
### 1. Happens-before
Если операция A happens-before операции B, то все изменения, сделанные в A, будут видны в B.
Примеры:
1. Разблокировка synchronized happens-before последующей блокировки того же монитора.
2. Запись в volatile happens-before чтения этой переменной.
3. Завершение потока happens-before Thread.join().
