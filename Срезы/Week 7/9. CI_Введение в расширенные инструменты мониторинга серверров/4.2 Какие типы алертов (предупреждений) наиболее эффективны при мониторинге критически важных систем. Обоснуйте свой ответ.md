# ✅ **1. Алерты на доступность (Availability Alerts)**
**Что это:** уведомления о том, что сервис недоступен или отвечает слишком долго.  
**Примеры:**
- Сервис возвращает 5xx ошибки
- Время ответа API > N секунд
- Под не поднялся после деплоя
**Почему важно:**  
Это **самый критичный** тип алерта — когда сервис недоступен, бизнес теряет деньги или данные прямо сейчас.
# ✅ **2. Алерты на деградацию производительности (Performance Degradation Alerts)**
**Что это:** метрики, показывающие, что система начинает “умирать”, но еще работает.  
**Примеры:**
- Увеличение latency
- Рост количества очередей/обработчиков
- Увеличение времени выполнения SQL-запросов
**Почему важно:**  
Позволяет обнаружить проблему **до сбоя**, предотвратить даунтайм.
# ✅ **3. Алерты на ресурсы (Resource Alerts)**
**Что это:** мониторинг CPU, RAM, диска.  
**Примеры:**
- CPU > 90% в течение 5 минут
- Heap usage > 80%
- Оставшийся диск < 1GB
**Почему важно:**  
Чаще всего падение происходит из-за нехватки ресурсов: OutOfMemory, заполнение диска, CPU starvation.
# ✅ **4. Алерты на ошибки приложений (Application Error Alerts)**
**Что это:** отслеживание количества ошибок в логах или счетчиков ошибок.  
**Примеры:**
- Резкий рост количества 4xx/5xx
- Рост количества exceptions в Java-приложении
- Ошибки подключения к БД
**Почему важно:**  
Часто именно ошибки в приложении предшествуют падению сервиса.
# ✅ **5. Алерты на состояние зависимостей (Dependency Alerts)**
Приложение может работать отлично, но зависимость может “лежать”.  
**Примеры:**
- Нет ответа от базы данных
- Kafka не принимает сообщения
- Redis перестал отвечать
**Почему важно:**  
Позволяет быстро локализовать проблему — не в сервисе, а во внешнем компоненте.
# ✅ **6. Алерты на нарушение бизнес-индикаторов (Business KPIs Alerts)**
Это самый зрелый уровень мониторинга.  
**Примеры:**
- Количество покупок/транзакций упало в 2 раза
- Очередь заказов стала пустой
- Слишком мало успехов и слишком много отказов операций
**Почему важно:**  
Такие алерты отражают **реальную боль бизнеса**, не только технические проблемы.
# ✅ **7. Алерты на события безопасности (Security Alerts)**
**Примеры:**
- Подозрительная активность логинов
- Непредвиденное изменение конфигурации
- Доступ к сервисам с неизвестных IP
**Почему важно:**  
Позволяет предотвратить атаки и утечку данных.
# ⭐ ИТОГО — самые важные алерты
Если выбирать топ-3 для критичных систем:

|Тип алерта|Почему критично|
|---|---|
|**Доступность**|Бизнес-простой, деньги теряются прямо сейчас|
|**Производительность**|Помогает предвидеть сбой заранее|
|**Ошибка/зависимости**|Частые причины сбоев, важно реагировать быстро|
