# Pull
**Git pull** - загружает изменения из удаленного репозитория и автоматически сливает (*merge*) их с вашей локальной веткой. Это стандартный способ синхронизировать ваш код с последними обновлениями.
## Как работает git pull?
Фактически, **git pull** = *git fetch + git merge*:
1. *git fetch* - скачивает новые коммиты из удаленного репозитория.
2. *git merge* - объединяет
### Чем git pull отличается от git fetch?

| git fetch                     | git pull                      |
| ----------------------------- | ----------------------------- |
| Только загружает изменения    | Загружает и сливает изменения |
| Безопасен (не меняет ваш код) | Может вызвать конфликты       |
| Требует ручного merge/rebase  | Делает слияние автоматически  |

## Основное использование
### 1. Стандартный pull (слияние изменений)
```bash
git pull origin имя_ветки
```
Пример:
```bash
git pull origin main
```
### 2. Pull с перезаписью локальных изменений
```bash
git pull --force   # или git reset --hard origin/main
```
⚠️ Используйте только если хотите стереть свои локальные правки!
### 3. Pull с rebase (для чистой истории)
```bash
git pull --rebase
```
Этот вариант:
- Не создает лишних merge-коммитов
- Ставит ваши коммиты поверх новых изменений.
## Полезные опции

| Опции       | Описание                                               |
| ----------- | ------------------------------------------------------ |
| --rebase    | Подтянуть изменения и перебазировать ваши коммиты      |
| --no-commit | Слить изменения, но не создавать коммит автоматически  |
| -ff-only    | Обновить ветку, только если возможен fast-forward      |
| --autostash | Временно сохранит незакоммиченные изменения перед pull |
## Примеры
### 1. Обычная синхронизация с main
```bash
git checkout main
git pull origin main
```
### 2. Обновление feature-ветки с rebase
```bash
git checkout my-feature
git pull --rebase origin main
```
### 3. Безопасный pull (если есть локальные изменения)
```bash
git stash          # Временно сохранить правки
git pull           # Обновиться
git stash pop      # Вернуть изменения
```
## Частые ошибки
### 1. Конфликты слияния
```bash
CONFLICT (content): Merge conflict in file.txt
```
✅**Решение:**
1. Открыть файл и исправить конфликт (ищется между <<<<<<<< и >>>>>>).
2. Добавить исправленный файл:
```bash
git add file.txt
```
3. Завершить слияние:
```bash
git commit
```
### 2. Ошибка: "Not a git repository"
```bash
fatal: not a git repository (on any parent up to mount point /)
```
✅**Решение:** Выполнить команду в папке с репозиторием.
### 3. Ошибка доступа
```bash
Permission denied (publickey).
```
**✅Решение:** Настроить SSH-ключ или использовать HTTPS.
## Советы
1. Всегда делайте **git pull** перед началом работы - это уменьшит конфликты.
2. Используйте **git pull** --rebase для линейной истории (если команда согласна)
3. Проверяйте изменения после **pull**:
```bash
git log --oneline --graph
```
