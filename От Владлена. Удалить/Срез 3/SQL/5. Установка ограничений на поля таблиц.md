### 1. Что такое первичный ключ в реляционной базе данных?

Первичный ключ — это уникальный идентификатор каждой строки в таблице. Он гарантирует, что значения в этом столбце (или комбинации столбцов) уникальны и не содержат NULL. Обычно используется для однозначной идентификации записей.

---

### 2. Какое SQL-определение используется для установки ограничения NOT NULL на поле таблицы?

При создании или изменении таблицы можно указать:


```sql
столбец тип_данных NOT NULL
```

или после создания таблицы:


```sql
ALTER TABLE таблица ALTER COLUMN столбец SET NOT NULL;
```

---

### 3. Как в SQL создать таблицу с автоматически инкрементируемым полем ID?

Зависит от СУБД:

- MySQL:


```sql
CREATE TABLE таблица (
    id INT AUTO_INCREMENT PRIMARY KEY,
    ...
);
```

- PostgreSQL:


```sql
CREATE TABLE таблица (
    id SERIAL PRIMARY KEY,
    ...
);
```

- SQL Server:


```sql
CREATE TABLE таблица (
    id INT IDENTITY(1,1) PRIMARY KEY,
    ...
);
```

---

### 4. Какое ограничение обеспечивает уникальность значений в столбце таблицы базы данных?

UNIQUE — ограничение, которое гарантирует, что все значения в столбце (или группе столбцов) уникальны:


```sql
ALTER TABLE таблица ADD CONSTRAINT имя_ограничения UNIQUE (столбец);
```

---

### 5. Как в SQL задать ограничение, чтобы значение поля соответствовало определенному диапазону чисел?

Используйте CHECK:


```sql
ALTER TABLE таблица ADD CONSTRAINT имя_ограничения CHECK (столбец BETWEEN нижняя_граница AND верхняя_граница);
```

или при создании таблицы:


```sql
столбец тип_данных CHECK (столбец BETWEEN нижняя_граница AND верхняя_граница)
```

---

### 6. Как реализовать внешний ключ в SQL, указывающий на первичный ключ в другой таблице?

Пример при создании таблицы:


```sql
FOREIGN KEY (столбец) REFERENCES другая_таблица(первичный_ключ);
```

или при создании всей таблицы:


```sql
CREATE TABLE таблица (
    ...
    внешний_id INT,
    CONSTRAINT fk_внешний FOREIGN KEY (внешний_id) REFERENCES другая_таблица(первичный_ключ)
);
```

---

### 7. Как в SQL использовать ограничение CHECK для определения сложных условий на значения в столбце?

Пример: ограничить возраст от 18 до 65 лет:


```sql
ALTER TABLE таблица ADD CONSTRAINT возраст_ограничение CHECK (возраст >= 18 AND возраст <= 65);
```

или при создании:


```sql
возраст INT CHECK (возраст >= 18 AND возраст <= 65)
```

---

### 8. Как реализовать составной первичный ключ в SQL?

Объявите несколько столбцов как первичный ключ:


```sql
PRIMARY KEY (столбец1, столбец2)
```

например:


```sql
CREATE TABLE заказ (
    заказ_id INT,
    товар_id INT,
    количество INT,
    PRIMARY KEY (заказ_id, товар_id)
);
```

---

### 9. Как в SQL ограничить доступ к определенным строкам таблицы с использованием политик безопасности?

Это зависит от системы управления базами данных и возможностей политики безопасности:

- В PostgreSQL можно использовать Row Level Security (RLS):


```sql
ALTER TABLE таблица ENABLE ROW LEVEL SECURITY;
CREATE POLICY политика ON таблица FOR SELECT TO роль USING (условие);
```

- В других системах могут использоваться представления с фильтрацией или механизмы контроля доступа.
