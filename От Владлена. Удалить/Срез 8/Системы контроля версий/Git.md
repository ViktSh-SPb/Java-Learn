### Какая команда используется для клонирования репозитория из удаленного источника?

 ```bash
  git clone <URL-репозитория>
```

Эта команда создает локальную копию удаленного репозитория.

### Какая команда используется для просмотра истории коммитов?

```bash
   git log
      ```

Показывает список всех коммитов с их хешами, авторами и сообщениями.

### Какая команда используется для создания новой ветки в Git?

```bash
   git branch имя_ветки    
   ```

Создает новую ветку, не переключаясь на нее.

### Какая команда позволяет сливать изменения из одной ветки в другую?

```bash
   git merge имя_ветки
   ```

Объединяет указанную ветку в текущую.

### Как можно отменить изменения в файле, возвращая его к последнему состоянию в коммите?

```bash
   git checkout -- путь/к/файлу
   ```

Восстанавливает файл до состояния последнего коммита, отменяя незакоммиченные изменения.

### Какая команда позволяет изменить последний коммит без изменения его содержимого (например, для исправления сообщения коммита)?

```bash
   git commit --amend   
    ```

Позволяет отредактировать сообщение последнего коммита или внести изменения.

### Какая команда позволяет перебазировать текущую ветку на указанную?

```bash
   git rebase <ветка_для_перебазирования>
   ```

Переносит текущие изменения на основание другой ветки.

### Как можно временно сохранить изменения, не коммитя их, для последующего применения?

```bash
   git stash
   ```

Сохраняет текущие незакоммиченные изменения в "тайник" и очищает рабочее пространство.

### Какой командой можно просмотреть список всех веток, включая удаленные, и последние коммиты в них?

```bash
   git branch -a -v
   ```

Показывает все локальные и удаленные ветки с информацией о последних коммитах.
