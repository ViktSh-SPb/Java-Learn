В Maven добавление зависимости от сторонней библиотеки делается через секцию `<dependencies>` в файле **`pom.xml`**.
## 1️⃣ Основная структура зависимости
```xml
<dependencies>
    <dependency>
        <groupId>group-id</groupId>
        <artifactId>artifact-id</artifactId>
        <version>version</version>
        <!-- scope можно опционально указать -->
    </dependency>
</dependencies>
```
**Пояснение полей:**

|Поле|Описание|
|---|---|
|`groupId`|Группа/организация библиотеки (например, `org.springframework.boot`)|
|`artifactId`|Название библиотеки (например, `spring-boot-starter-web`)|
|`version`|Версия библиотеки (например, `3.3.0`)|
|`scope`|Опционально, область использования: `compile` (по умолчанию), `test`, `provided`, `runtime`|
## 2️⃣ Пример: добавление Spring Boot
```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
        <version>3.3.0</version>
    </dependency>
</dependencies>
```
- Maven автоматически скачает библиотеку из **Maven Central**.
- Она будет доступна при компиляции и запуске проекта.
## 3️⃣ Добавление зависимости только для тестов
```xml
<dependencies>
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter</artifactId>
        <version>5.10.0</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```
- С библиотекой можно работать **только в тестах** (`src/test/java`).
## 4️⃣ Добавление библиотеки из кастомного репозитория
Если библиотека **не находится в Maven Central**, нужно указать репозиторий:
```xml
<repositories>
    <repository>
        <id>my-repo</id>
        <url>https://example.com/maven2</url>
    </repository>
</repositories>

<dependencies>
    <dependency>
        <groupId>com.example</groupId>
        <artifactId>custom-lib</artifactId>
        <version>1.0.0</version>
    </dependency>
</dependencies>
```
## 5️⃣ Проверка
После добавления зависимости можно выполнить команду:
```bash
mvn clean install
```
- Maven скачает зависимость в локальный кэш (`~/.m2/repository`)
- Сборка проекта будет использовать новую библиотеку.
### ✅ Резюме
1. Открываем `pom.xml`.
2. Добавляем `<dependency>` с `groupId`, `artifactId`, `version`.
3. При необходимости указываем `scope` или `<repositories>` для нестандартных репозиториев.
4. Выполняем `mvn install` или `mvn compile` для проверки.
