## Виды ThreadPool
**1. FixedThreadPool**  
Создаётся с фиксированным числом потоков через `Executors.newFixedThreadPool(n)`.
- Все потоки фиксированы, если все заняты — новые задачи ждут в очереди.
- Используется, когда известно количество задач и нужно ограничить число одновременно работающих потоков.
**2. CachedThreadPool**  
Создаётся через `Executors.newCachedThreadPool()`.
- Пул динамически создаёт новые потоки при необходимости и переиспользует свободные.
- Потоки, не использующиеся больше 60 секунд, уничтожаются.
- Подходит для большого числа коротких асинхронных задач с непредсказуемым количеством.
**3. SingleThreadExecutor**  
Создаётся через `Executors.newSingleThreadExecutor()`.
- Всегда один поток, все задачи выполняются последовательно.
- Используется, когда важно, чтобы задачи выполнялись строго по очереди, но не в основном потоке.
**4. ScheduledThreadPool**  
Создаётся через `Executors.newScheduledThreadPool(n)`.
- Позволяет запускать задачи с задержкой или периодически (`schedule()`, `scheduleAtFixedRate()`).
- Отлично подходит для таймеров, планировщиков и периодических задач.
**5. WorkStealingPool** (Java 8+)  
Создаётся через `Executors.newWorkStealingPool()`.
- Потоки динамически балансируют нагрузку между собой: свободные потоки «воруют» задачи у занятых.
- Идеально для параллельных задач, которые можно разбить на подзадачи.
