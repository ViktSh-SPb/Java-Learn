## 1️⃣ Инкрементальная сборка (Incremental Build)
**Gradle:**
- Каждая задача (`task`) знает свои **входные файлы (inputs)** и **выходные результаты (outputs)**.
- Gradle сохраняет **хэши файлов** и повторно выполняет только те задачи, где есть изменения.
- Если код модуля не изменился, Gradle пропустит его сборку и тесты (UP-TO-DATE).
**Maven:**
- По умолчанию пересобирает весь модуль при любой сборке (`mvn clean install`).
- Нет встроенного механизма инкрементальной сборки (только через сторонние плагины).
**Влияние:** для больших проектов Gradle может сократить сборку с минут до секунд.
## 2️⃣ Build Cache (локальный и удалённый кэш)
**Gradle:**
- **Local Build Cache** — кэширует результаты задач на локальной машине.
- **Remote Build Cache** — можно использовать общий кэш в CI/CD, чтобы задачи, выполненные на другом сервере, не пересобирались заново.
**Maven:**
- Кэширует только зависимости, но **не результаты сборки**.
- Каждая сборка на другой машине требует пересборки артефактов.
**Влияние:** Gradle минимизирует повторную работу, ускоряя CI/CD и локальные сборки.
## 3️⃣ Конфигурационный кэш (Configuration Cache)
**Gradle:**
- Кэширует **фазу конфигурации проекта** (анализ `build.gradle`, создание графа задач).
- При повторной сборке Gradle может сразу перейти к выполнению задач без повторного анализа проекта.
**Maven:**
- Всегда заново анализирует POM и зависимости.
**Влияние:** особенно заметно в больших многомодульных проектах, где конфигурация может занимать десятки секунд.
## 4️⃣ Параллельная сборка
**Gradle:**
- Поддерживает `--parallel` для выполнения задач из **разных модулей одновременно**, если нет зависимостей.
- Автоматически строит DAG (Directed Acyclic Graph) задач и выполняет их параллельно.
**Maven:**
- Имеет `-T` (threads), но параллельность ограничена модульной структурой и сложнее в настройке.
- Меньше контроля над зависимостями задач.
## 5️⃣ Гибкая настройка и уменьшение лишних действий
**Gradle:**
- Можно легко создавать кастомные задачи и исключать ненужные шаги.
- Поддерживает **динамическое конфигурирование** зависимостей и условий выполнения задач.
**Maven:**
- Фиксированные фазы и жизненный цикл; кастомизация сложнее.
**Влияние:** меньше «лишней работы» → быстрее сборка.
## 6️⃣ Пример сравнения времени сборки

|Сценарий|Maven|Gradle|
|---|---|---|
|Полная сборка большого проекта|3–5 мин|2–3 мин|
|Пересборка одного изменённого модуля|1–2 мин|5–10 сек (UP-TO-DATE)|
|CI/CD с кэшированием артефактов|Нет|<10 сек для неизменённых задач|
### ✅ Краткий вывод
**Преимущества Gradle по производительности:**
1. Инкрементальная сборка → пересобираются только изменённые части.
2. Build Cache → повторно используемые результаты задач, локально и в CI.
3. Конфигурационный кэш → ускоряет фазу конфигурации.
4. Параллельная сборка → эффективнее для многомодульных проектов.
5. Гибкая настройка → исключение лишних действий.
> Gradle особенно выигрывает на **больших проектах с множеством модулей и зависимостей**, где Maven будет выполнять много лишней работы.
