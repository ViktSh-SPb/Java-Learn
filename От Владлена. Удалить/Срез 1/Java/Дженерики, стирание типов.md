### Что такое дженерики в Java?

- Дженерики — это механизм в Java, позволяющий создавать обобщённые классы, интерфейсы и методы, которые работают с параметризованными типами данных. Это обеспечивает типобезопасность и повторное использование кода.

### Для чего используются дженерики в Java?

- Для повышения безопасности типов во время компиляции.
- Для избежания приведения типов и ошибок времени выполнения.
- Для создания универсальных и переиспользуемых компонентов, например, коллекций.

### Как объявить класс или интерфейс с использованием дженериков?

```java
// Объявление обобщенного класса
public class Box<T> {
    private T item;

    public void set(T item) {
        this.item = item;
    }

    public T get() {
        return item;
    }
}

// Объявление обобщенного интерфейса
public interface Container<T> {
    void add(T item);
    T remove();
}
````

### Как работает механизм стирания типов в Java?

- Во время компиляции информация о параметрах типа (дженериках) стирается, то есть заменяется на их границы или `Object`, если границы не указаны.
- В результате во время выполнения типы не сохраняются, что может привести к некоторым ограничениям и необходимости приведения типов.

### Приведите пример параметризованного типа с использованием дженериков.

```java
List<String> list = new ArrayList<>();
list.add("Hello");
String s = list.get(0); // Нет необходимости приведения типа
```

### Как ограничить типы, которые можно использовать в дженериках?

- Используйте ограничения с помощью ключевого слова `extends`:

```java
public class NumberBox<T extends Number> {
    private T number;

    public void set(T number) {
        this.number = number;
    }

    public T get() {
        return number;
    }
}
```

Это ограничит типы `T` только классами, наследующими `Number`.

### Какие проблемы решает механизм стирания типов и какие проблемы он создает?

Проблемы, которые решает:

- Позволяет использовать один и тот же байт-код для разных типов данных.
- Обеспечивает совместимость с существующим кодом без изменений.

Проблемы, которые создает:

- Потеря информации о конкретных типах во время выполнения.
- Невозможность создавать массивы параметризованных типов (`new T[]` невозможно).
- Необходимость приведения типов при использовании необобщенных методов или коллекций.

### Как реализовать дженерик метод, который принимает любой тип данных?

```java
public <T> void printItem(T item) {
    System.out.println(item);
}
```

Этот метод может принимать аргумент любого типа.

### Каковы особенности использования дженериков при работе с исключениями и перегруженными методами?

- Дженерики не могут быть использованы с проверяемыми исключениями из-за ограничений компилятора.
- Перегруженные методы с разными дженериками могут привести к проблемам из-за стирания типов — компилятор может считать их одинаковыми.
- Обычно избегают использования дженериков в сигнатурах методов, выбрасывающих исключения, чтобы избежать неоднозначности.
