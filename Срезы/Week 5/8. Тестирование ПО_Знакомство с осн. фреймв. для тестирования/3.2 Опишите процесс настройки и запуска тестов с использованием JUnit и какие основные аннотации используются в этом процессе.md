## 1️⃣ Настройка проекта
1. **Добавление зависимости JUnit**:
**Maven**
```xml
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>
```
**Gradle**
```gradle
testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
```
2. **Создание структуры каталогов**:
```
src/main/java/...  # основной код
src/test/java/...  # тесты
```
## 2️⃣ Создание тестового класса
- Для каждого класса приложения создаётся отдельный тестовый класс.
- Обычно имя тестового класса = имя тестируемого класса + `Test`.
```java
class CalculatorTest {
    // тестовые методы здесь
}
```
## 3️⃣ Написание тестовых методов
- Метод помечается аннотацией `@Test`.
- Внутри метода: подготовка данных → вызов тестируемого метода → проверка результата с помощью assertions.
```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calc = new Calculator();
        int result = calc.add(2, 3);
        assertEquals(5, result);
    }
}
```
## 4️⃣ Основные аннотации JUnit

|Аннотация|Назначение|
|---|---|
|`@Test`|Помечает метод как тестовый|
|`@BeforeEach`|Выполняется перед каждым тестовым методом (инициализация объектов)|
|`@AfterEach`|Выполняется после каждого тестового метода (очистка ресурсов)|
|`@BeforeAll`|Выполняется один раз перед всеми тестами класса (статический метод)|
|`@AfterAll`|Выполняется один раз после всех тестов класса (статический метод)|
|`@Disabled`|Отключает тест (например, временно)|
|`@ParameterizedTest`|Позволяет запускать один тест с разными входными данными|
## 5️⃣ Настройка и очистка окружения
- `@BeforeEach` и `@AfterEach` используются для подготовки и очистки данных для каждого теста.
- `@BeforeAll` и `@AfterAll` — для общей подготовки или освобождения ресурсов, например, подключения к базе данных.
```java
@BeforeEach
void setUp() {
    calculator = new Calculator();
}

@AfterEach
void tearDown() {
    calculator = null;
}
```
## 6️⃣ Запуск тестов
- **В IDE:** через контекстное меню тестового класса или метода.
- **Через Maven/Gradle:**
```bash
mvn test
gradle test
```
- Test Runner выполняет:
    1. `@BeforeAll` (один раз на класс)
    2. `@BeforeEach` (перед каждым тестом)
    3. Тестовый метод `@Test`
    4. `@AfterEach` (после каждого теста)
    5. `@AfterAll` (один раз после всех тестов)
- После выполнения формируется **отчёт с результатами**: успешно / неуспешно / пропущено.
## 7️⃣ Итоговый процесс
```
1. Настройка проекта (добавление JUnit)
2. Создание тестового класса
3. Написание тестовых методов (@Test)
4. Подготовка окружения (@BeforeEach, @BeforeAll)
5. Очистка ресурсов (@AfterEach, @AfterAll)
6. Запуск тестов (IDE / Maven / Gradle)
7. Анализ результатов и исправление кода
```
