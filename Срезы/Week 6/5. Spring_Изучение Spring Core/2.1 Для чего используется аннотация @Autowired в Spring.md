Аннотация **`@Autowired`** в **Spring Framework** используется для **автоматического внедрения зависимостей (Dependency Injection, DI)** - то есть, чтобы Spring **автоматически предоставил нужный объект (bean)** в место, где он требуется.
## 🔧 Основная идея
Когда в классе вы указываете зависимость (например, сервис, репозиторий, компонент и т.д.), Spring сам найдет подходящий bean (объект, управляемый контейнером Spring) и подставит его, без необходимости создавать объект вручную через `new`.
## 🧩 Пример
```java
@Service
public class OrderService {

    @Autowired
    private PaymentService paymentService;

    public void processOrder() {
        paymentService.pay();
    }
}
```
➡️ Здесь:
- `PaymentService` - это зависимость.
- Spring при старте приложения ищет bean типа `PaymentService` (например, класс, помеченный `@Service`).
- Находит его и автоматически внедряет в `OrderService`.
## 📦 Где можно использовать `@Autowired`
1. **На поле (field injection)**
    ```java
    @Autowired
    private UserRepository userRepository;
    ```
    ➤ Прямое внедрение в поле класса (просто, но менее тестируемо).
2. **На конструкторе (constructor injection)** - **рекомендуемый способ**
    ```java
    @Service
    public class UserService {
    
        private final UserRepository repository;
    
        @Autowired
        public UserService(UserRepository repository) {
            this.repository = repository;
        }
    }
    ```
    ➤ Безопаснее, потому что зависимость передаётся при создании объекта (можно сделать `final`, проще тестировать).
3. **На сеттере (setter injection)**
    ```java
    private EmailService emailService;
    
    @Autowired
    public void setEmailService(EmailService emailService) {
        this.emailService = emailService;
    }
    ```
    ➤ Гибкий вариант, если зависимость может быть изменена или не обязательна.
## ⚙️ Как работает
- При запуске Spring создаёт **ApplicationContext**, который управляет всеми bean’ами.
- Когда Spring видит `@Autowired`, он:
    1. Ищет bean подходящего типа.
    2. Если найден один - внедряет его.
    3. Если найдено несколько — выдаёт ошибку, **если не указано, какой выбрать**.
## 🧭 Если есть несколько кандидатов
Можно уточнить, какой bean нужно внедрить:
- **`@Qualifier("beanName")`**
    ```java
    @Autowired
    @Qualifier("paypalPaymentService")
    private PaymentService paymentService;
    ```
- Или пометить нужный bean как **@Primary**:
    ```java
    @Service
    @Primary
    public class DefaultPaymentService implements PaymentService { ... }
    ```
## ✅ Преимущества
- Избавляет от явного создания объектов (`new`).
- Позволяет легко заменять реализации (например, mock при тестировании).
- Поддерживает гибкость и слабую связанность компонентов.
## ⚠️ Важно помнить

|Особенность|Описание|
|---|---|
|Не нужно использовать `new`|Spring создаёт и внедряет объекты сам|
|По умолчанию внедрение по типу|Если несколько — требуется `@Qualifier`|
|Для необязательных зависимостей|Можно указать `@Autowired(required = false)`|
|Лучше использовать конструктор|Безопаснее и упрощает тестирование|
### 💡 Кратко:
> **`@Autowired`** — это аннотация, которая говорит Spring:  
> «Подставь мне сюда нужный объект (bean) из контекста, я сам не буду его создавать».
