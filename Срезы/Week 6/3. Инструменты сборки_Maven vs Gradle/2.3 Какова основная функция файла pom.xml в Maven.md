Файл **`pom.xml`** (Project Object Model) — это **центральный конфигурационный файл проекта в Maven**.
Его основная функция:
> **Определить проект и его зависимости, настройки сборки, плагины и метаданные, чтобы Maven мог автоматически собрать, протестировать и упаковать проект.**
## 1️⃣ Идентификация проекта
В `pom.xml` задаются уникальные координаты проекта:
```xml
<groupId>com.example</groupId>  <!-- группа/организация -->
<artifactId>my-app</artifactId> <!-- название артефакта -->
<version>1.0.0</version>       <!-- версия -->
<packaging>jar</packaging>     <!-- тип артефакта: jar, war, pom -->
```
Эти данные используются для управления зависимостями и публикации артефактов в репозитории.
## 2️⃣ Управление зависимостями
`pom.xml` описывает внешние библиотеки, которые нужны проекту:
```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
        <version>3.3.0</version>
    </dependency>
</dependencies>
```
Maven автоматически скачивает эти зависимости из центрального репозитория.
## 3️⃣ Настройка сборки и плагинов
В `pom.xml` можно подключать плагины для компиляции, тестирования, упаковки и других задач:
```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.11.0</version>
            <configuration>
                <source>17</source>
                <target>17</target>
            </configuration>
        </plugin>
    </plugins>
</build>
```
## 4️⃣ Модули и управление проектом
Для многомодульных проектов `pom.xml` может определять дочерние модули:
```xml
<modules>
    <module>core</module>
    <module>web</module>
</modules>
```
## 5️⃣ Дополнительные метаданные
- Информация о разработчике, лицензии, URL проекта
- Репозитории для зависимостей
- Профили сборки (`<profiles>`) для разных сред (dev, prod)
### ✅ Кратко
**`pom.xml` — это “карта проекта” для Maven**, которая сообщает:
1. Кто вы и как называется проект
2. Какие зависимости нужны
3. Как собирать, тестировать и упаковывать проект
4. Какие модули и плагины использовать
Без `pom.xml` Maven просто не знает, что делать с проектом.
