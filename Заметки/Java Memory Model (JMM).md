# Java Memory Model (JMM)
**Java Memory Model** - это спецификация, которая описывает, как потоки взаимодействуют через память. Она задает правила: какие изменения, сделанные одним потоком, будут видны другим, управляет кэшированием, переупорядочиванием инструкций, видимостью переменных.
## Основные гарантии JMM
### 1. Happens-before
Если операция **A** *happens-before* операции **B**, то все изменения, сделанные в **A**, будут видны в **B**.
Примеры:
1. Разблокировка *synchronized happens-before* последующей блокировки того же монитора.
2. Запись в *volatile happens-before* чтения этой переменной.
3. Завершение потока *happens-before Thread.join()*.
### 2. Атомарность
1. Чтение / запись переменных размером не больше 32 бит (*boolean, byte, short, char, int, reference*) всегда атомарны.
2. Для *long* и *double* атомарность гарантируется с **Java 5**.
### 3. Видимость
1. Без *volatile* или синхронизации изменения могут быть не видны другим потокам (потому что могут остаться в кэше процессора).
2. *volatile, synchronized, final* помогают установить правильные границы видимости.
### 4. Переупорядочивание инструкций (Reordering)
1. **JIT** и процессор могут менять порядок выполнения инструкций для оптимизации.
2. **JMM** определяет, какие переупорядочивания допустимы, чтобы не нарушить корректность многопоточных программ.
