Основные **компоненты системы непрерывной интеграции (CI)** — это набор инструментов и процессов, которые обеспечивают автоматическую сборку, тестирование и анализ кода при каждом изменении.
## ✅ **1. Репозиторий исходного кода (VCS)**
Главное хранилище проекта, где разработчики сохраняют изменения.  
Используются:
- **GitHub**
- **GitLab**
- **Bitbucket**
- **Azure Repos**
Все процессы CI начинаются с события в репозитории (push, merge request / pull request).
## ✅ **2. CI-сервер (оркестратор сборок)**
Инструмент, который запускает сборки и координирует выполнение шагов.  
Примеры:
- **Jenkins**
- **TeamCity**
- **GitHub Actions**
- **GitLab CI/CD**
- **Azure DevOps Pipelines**
CI-сервер:
- отслеживает изменения в коде,
- запускает pipeline,
- собирает результаты тестов,
- публикует артефакты.
## ✅ **3. Build-агенты (исполнители задач)**
Отдельные узлы (агенты/раннеры), которые выполняют сборку и тестирование.
Бывают:
- **static agents** (всегда запущены)
- **dynamic agents** (создаются под задачу, часто в Docker или Kubernetes)
Агенты выполняют:
- сборку проекта,
- прогон тестов,
- генерацию артефактов,
- пуш результатов.
## ✅ **4. Pipeline (конвейер сборки)**
Последовательность шагов, которые описывают CI-процесс:
- checkout кода
- установка зависимостей
- сборка проекта
- запуск тестов
- статический анализ
- публикация артефактов
Обычно описывается в YAML-файле:
- GitLab: `.gitlab-ci.yml`
- GitHub: `.github/workflows/*.yml`
- Jenkins: `Jenkinsfile`
## ✅ **5. Система управления зависимостями**
Инструменты, которые гарантируют воспроизводимую сборку:
- Maven / Gradle (Java)
- npm / yarn (JS)
- pip / poetry (Python)
- go modules
Важно, чтобы сборки были детерминированными.
## ✅ **6. Набор авто-тестов**
Код должен проходить автоматизированные тесты:
- unit-test
- integration-test
- smoke-test
Тесты запускаются при каждом коммите.
## ✅ **7. Хранилище артефактов**
Место для собранных пакетов или результатов тестов:
- Docker Registry
- Nexus / Artifactory
- GitLab Package Registry
- S3
CI-сервер публикует туда результаты сборки.
## ✅ **8. Система уведомлений**
CI уведомляет разработчиков о результатах:
- Slack
- Email
- Telegram
- GitHub/GitLab уведомления
Это помогает быстро реагировать на ошибки.
## Итоговая схема компонентов CI
```
Developer -> Git Push -> VCS -> CI Server -> Build Agent -> Tests -> Artifacts -> Notifications
```
