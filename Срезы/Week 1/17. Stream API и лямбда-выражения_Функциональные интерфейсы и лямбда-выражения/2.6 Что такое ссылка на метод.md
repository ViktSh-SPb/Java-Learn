**Ссылка на метод (method reference)** — это **краткая форма лямбда-выражения**, которая _неявно_ вызывает существующий метод.  
Она используется там, где лямбда **просто вызывает один метод** без дополнительной логики.
## 1. Общая идея
Лямбда:
```java
x -> System.out.println(x)
```
Ссылка на метод:
```java
System.out::println
```
Обе формы **эквивалентны** и реализуют один и тот же **функциональный интерфейс**.
## 2. Общий синтаксис
```java
Класс::метод
объект::метод
Класс::new
```
## 3. Виды ссылок на методы
### 1️⃣ Ссылка на **static-метод**
```java
Integer::parseInt
```
Эквивалент:
```java
s -> Integer.parseInt(s)
```
Использование:
```java
Function<String, Integer> f = Integer::parseInt;
```
### 2️⃣ Ссылка на метод **конкретного объекта**
```java
System.out::println
```
Эквивалент:
```java
s -> System.out.println(s)
```
Использование:
```java
Consumer<String> printer = System.out::println;
```
### 3️⃣ Ссылка на **нестатический метод класса**
(самый «магический» вариант)
```java
String::toUpperCase
```
Эквивалент:
```java
s -> s.toUpperCase()
```
Использование:
```java
Function<String, String> f = String::toUpperCase;
```
⚠ Первый параметр функционального интерфейса становится `this`.
### 4️⃣ Ссылка на **конструктор**
```java
ArrayList::new
```
Эквивалент:
```java
() -> new ArrayList<>()
```
Использование:
```java
Supplier<List<String>> supplier = ArrayList::new;
```
## 4. Method reference и функциональный интерфейс
Method reference **всегда приводится** к функциональному интерфейсу:
```java
Runnable r = System.out::println; // ❌ не скомпилируется
Runnable r = () -> System.out.println(); // ✔
```
Компилятор сопоставляет:
- сигнатуру метода
- сигнатуру абстрактного метода интерфейса
## 5. Когда использовать ссылку на метод
✔ лямбда просто вызывает один метод  
✔ нет дополнительной логики  
✔ повышает читаемость
❌ если внутри есть условия, несколько вызовов, side effects
## 6. Частые ошибки
- несоответствие сигнатур
- попытка использовать ссылку без функционального интерфейса
- путаница между `obj::method` и `Class::method`
## 7. Коротко
> Ссылка на метод — это сокращённая форма лямбда-выражения, которая указывает на существующий метод или конструктор и используется как реализация функционального интерфейса.
