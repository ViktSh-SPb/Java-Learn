### Что такое статическое связывание в Java?

- Статическое связывание — это процесс определения вызова метода во время компиляции. Обычно используется для вызова статических методов, методов с одинаковой сигнатурой и при использовании методов перегрузки.

### Какие типы методов обычно используют статическое связывание в Java?

- Статические методы (`static`)
- Методы с одинаковой сигнатурой при перегрузке
- Методы, вызванные через переменные типа класса или интерфейса, где конкретная реализация известна на этапе компиляции

### Что такое динамическое связывание в Java?

- Динамическое связывание — это процесс определения вызываемого метода во время выполнения программы (во время выполнения). Используется для вызова методов экземпляров, особенно при использовании наследования и полиморфизма.

---

### Приведите пример, когда используется динамическое связывание в Java.

```java
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog();
        myDog.sound(); // вызовется метод Dog.sound() благодаря динамическому связыванию
    }
}
````

### Как работает механизм динамического связывания при наследовании?

- Когда вызывается метод через ссылку на базовый класс, JVM определяет конкретную реализацию метода во время выполнения, основываясь на реальном типе объекта. Это обеспечивает полиморфизм: вызывается тот метод, который реализован в реальном классе объекта.

### Чем отличается статическое связывание от динамического на примере методов классов?

|Характеристика|Статическое связывание|Динамическое связывание|
|---|---|---|
|Время определения|Во время компиляции|Во время выполнения|
|Используемые методы|Статические, перегруженные|Обычные методы экземпляра|
|Пример|Вызов статического метода или перегруженного метода по имени и параметрам|Вызов переопределенного метода через ссылку на базовый класс|

### Как JVM определяет, какой метод вызывать при динамическом связывании?

- JVM использует таблицу виртуальных методов (виртуальную таблицу), которая содержит ссылки на конкретные реализации методов. При вызове метода через ссылку на базовый класс JVM ищет соответствующую реализацию в таблице для реального типа объекта.

### Каковы особенности статического связывания при работе с перегруженными методами?

- В случае перегрузки методов (одинаковое имя, разные параметры), выбор метода происходит во время компиляции по сигнатуре вызова.
- Вызов определяется по типу переменной и списку аргументов.

### Как влияет модификатор final на процесс динамического связывания методов?

- Методы с модификатором `final` не могут быть переопределены.
- Поэтому для `final` методов JVM использует статическое связывание — вызов происходит напрямую по имени метода без поиска в таблице виртуальных методов.
